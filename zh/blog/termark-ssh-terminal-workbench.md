# 如果你经常连服务器，可以试试 Termark

如果你的日常工作里经常出现这些场景：

打开终端连服务器；  
在多个标签页之间切换；  
找某台机器的账号、密钥、跳板机配置；  
临时上传一个配置文件，或者下载一份日志；  
在几台机器上重复执行同一条命令；  
看着一屏日志，想让 AI 帮忙分析一下，但又不希望它直接替你乱执行命令。

那你可能会需要一个更适合远程工作的桌面工具。

我做的这个工具叫 **Termark**，它是一个 SSH 终端管理工具，也可以理解成一个围绕服务器资产展开的远程工作台。

它不是只解决“连上服务器”这一步，而是把资产管理、终端会话、SFTP 文件传输、批量命令、端口转发、命令片段、会话记录、多设备同步和 AI 辅助这些常用动作放在一起。

## 为什么我觉得还需要一个新的 SSH 工具

SSH 命令本身已经很好用了。

如果只是偶尔登录一台服务器，系统终端加一条 `ssh user@host` 就够了。

但真实工作里，麻烦通常不是 SSH 协议本身，而是围绕 SSH 的一堆碎片：

- 服务器越来越多，分组、搜索、备注和凭证开始变重要。
- 有些机器要走跳板机，有些要走代理，有些还要兼容老旧算法。
- 排查问题时，一边看日志，一边传文件，一边开端口转发。
- 常用命令散落在笔记、聊天记录和历史命令里，每次都要重新找。
- 多台机器要执行同一条命令，复制粘贴很机械，也容易漏。
- 终端里出现过的关键输出，过一会儿想回看却找不到。
- 多台电脑切换时，资产、凭证、片段和设置不同步。

Termark 想解决的是这些“连接之后”的事情。

它的目标不是把每个功能都做得很复杂，而是把后端、运维、开发者和服务器重度用户每天都会用到的动作收在同一个地方，让远程工作少一点切换、少一点重复配置、少一点复制粘贴。

## 先把服务器资产管起来

Termark 打开后，最左侧是一棵资产树。

你可以按自己的习惯管理服务器，比如生产环境、测试环境、客户项目、数据库、网关、家里的 NAS。分组支持多级，也可以搜索、拖拽排序。

已有的 `~/.ssh/config` 可以直接导入，批量迁移时也可以从 CSV 导入。

现在 Termark 不只支持 SSH，也可以管理：

- SSH
- Telnet
- 串口
- 本地终端
- NextTerminal 资产

这点对我来说很重要。

很多团队或个人的环境并不“纯净”：有新服务器，也有老设备；有云主机，也有内网机器；有些资产已经放在 NextTerminal 里，有些只是本地 SSH 配置。Termark 希望提供一个统一入口，而不是要求你把所有东西换成同一种管理方式。

最近版本里，资产树也做了一些更贴近日常操作的细节：支持 Shift 范围多选、批量连接、展开分组菜单、复制 IP、图标对齐、自动识别部分操作系统 Logo。看起来都是小功能，但用久了会省很多动作。

## 终端本身要稳定，也要顺手

Termark 的终端基于 xterm.js。

基础能力包括多标签、分屏、搜索、复制会话、自动重连、选中复制、右键粘贴、主题、字体、快捷键和关键字高亮。

这些功能听起来不新鲜，但它们决定了一个终端工具能不能长期用。

比如我自己很常用这几个点：

- **命令片段**：把 Docker 清理、systemd 状态检查、磁盘检查、日志过滤、重启服务这些命令保存起来，不用每次重新敲。
- **关键字高亮**：把 `ERROR`、`WARN`、`failed`、`success`、接口耗时、业务关键词标出来，看日志时更容易抓重点。
- **复制会话**：从当前 SSH 会话复制一个新标签，最近版本还支持自动进入相同目录，排查问题时很方便。
- **本地终端**：本地终端标签标题会自动更新，也支持拖拽文件识别路径，日常使用更自然。

终端体验里有很多细节很难用一句宣传语讲清楚，但真正用起来会很明显。

比如标签页多的时候能不能滚动，拖拽排序会不会顺手，中文会不会乱码，复制粘贴在不同系统上是否稳定，重连后输出还在不在，快捷键设置是不是清楚。

Termark 最近几个版本花了不少时间打磨这些地方。

## 多台机器执行同一条命令，不用一个窗口一个窗口粘贴

有些操作天然就是批量的。

比如：

- 查看一组机器的磁盘空间。
- 检查某个服务是否都已经启动。
- 发布后确认多台服务器版本是否一致。
- 批量查看端口、进程、系统负载。
- 对一组机器执行相同的巡检命令。

传统做法通常是开很多终端窗口，一个个粘贴命令。

Termark 里可以选择多台资产，然后进入批量执行页面。每台机器会有独立的终端面板，你在顶部输入一次命令，就可以发送到所有目标机器。

输出不会混在一起，每台机器仍然独立显示。这样既能统一下发，又能分别观察结果。

如果你已经把常用命令保存成片段，也可以直接在批量执行里选择片段。

最近的资产树范围多选和批量连接，就是为了让这类场景更顺手。

## SFTP 放在终端旁边，传文件不用再切工具

远程工作里，文件传输很常见。

很多时候你不是专门要“管理文件”，只是排查过程中顺手要做一件事：

- 上传一份配置。
- 下载一段日志。
- 看一下目录结构。
- 新建或重命名文件夹。
- 修改权限。
- 批量下载排查材料。

Termark 内置了 SFTP 工作区，是双面板结构。左侧可以是本机，右侧可以是远程服务器，也可以在不同资产之间切换。

支持上传、下载、拖拽、新建文件夹、新建文件、重命名、删除、修改权限、批量下载、批量删除和传输任务查看。

在 SSH 终端里，也可以打开当前会话对应的 SFTP 标签页。

如果开启目录跟随，终端里切换目录后，SFTP 也能贴近当前工作目录。排查问题时，这比单独打开一个文件传输工具再手动找路径要省事很多。

最近版本也重构和优化了 SFTP 文件浏览器、重连、传输、拖拽预览、路径复制、长文件名显示等细节。

## 端口转发、跳板机、代理这些配置，也应该被保存下来

SSH 端口转发很好用，但命令不太好记。

尤其是需要经常访问内网数据库、后台服务、临时调试接口时，每次重新拼 `ssh -L` 或 `ssh -R` 都很烦。

Termark 支持本地转发和远程转发，可以把常用规则保存下来。

比如：

- 把远程 MySQL 映射到本机端口。
- 通过跳板机访问内网服务。
- 临时把本地调试服务暴露给远端机器。
- 调试只能在服务器网络里访问的接口。

凭证和连接配置也可以集中管理。

Termark 支持密码凭证、私钥凭证、密钥口令、多级跳板、HTTP 代理、Socks5 代理、连接超时、环境变量、Backspace 模式、编码配置和 SFTP Sudo 提权模式。

最近版本还补充了 SSH keyboard-interactive 认证、更多主机密钥算法、预连接认证和主机密钥确认，用来兼容更多真实环境里的服务器。

这些能力不会每天都被提起，但一旦遇到复杂网络环境，它们就很关键。

## AI 辅助：站在终端旁边，但控制权还在你手里

Termark 里有 AI 助手。

我不太想把它包装成“自动运维”。对服务器工具来说，自动执行命令是一件需要谨慎对待的事。

Termark 的 AI 更像是站在终端旁边的助手：它可以参考最近的终端输出，帮你分析报错、解释日志、整理下一步排查思路，也可以在你确认后执行一些命令。

你可以这样用它：

- 让它分析一段服务启动失败的日志。
- 让它根据当前输出判断下一步该查什么。
- 让它帮你写一条过滤日志或查看进程的命令。
- 让它解释某个 Linux 命令的参数。
- 让它在你确认后执行低风险排查命令。

AI 配置支持兼容 OpenAI 协议的接口，可以配置自己的 API 地址、Key 和模型。你可以接 OpenAI、DeepSeek、Qwen、Kimi、ollama 等兼容服务。

最近版本里，Termark 也增加了对 Codex 的支持，优化了模型切换、AI 设置、右键菜单入口和命令确认策略。

命令执行这件事上，Termark 会保留确认环节。你可以选择每条命令都确认，也可以只对危险命令确认。像删除、格式化磁盘、重启、关机、清空防火墙这类高风险命令，会被识别出来并要求确认。

我的想法很简单：AI 可以提高效率，但服务器的控制权必须仍然在用户手里。

## NextTerminal 用户也可以直接接进来

如果你已经在用 NextTerminal，Termark 也可以接入。

配置时通过浏览器授权，不需要手动创建 API Token。授权后，可以把 NextTerminal 里的 SSH 资产加载进 Termark 的资产树。

Termark 支持配置多个 NextTerminal 环境，也支持通过 WebSocket 隧道访问，以及上游访问认证，包括 Basic Auth 和 mTLS。最近版本还增加了 NextTerminal 上游代理设置。

这意味着你不需要在已有资产体系和本地桌面工具之间二选一。

资产可以继续由 NextTerminal 管理，日常连接、SFTP、批量执行、命令片段和 AI 辅助则可以在 Termark 里完成。

## 多设备同步，但服务端看不到明文

如果你在公司电脑、家里电脑、笔记本和台式机之间切换，资产和命令片段不同步会很影响体验。

Termark 支持多种同步方式：

- 官方同步服务
- WebDAV
- S3 兼容对象存储
- iCloud
- 本地目录

同步内容包括主机、凭证、命令片段和配置等数据。

这里我比较重视的是安全模型：同步数据会在客户端加密后再上传。同步密码不会发给服务端，服务端拿到的是密文。

换句话说，服务端只负责保存数据，不负责也不能解密你的服务器信息。

多设备同时修改时，Termark 会做版本冲突检测。如果发现冲突，会让你选择从云端恢复，或者用本地数据覆盖云端，而不是静默覆盖。

最近版本里，本地终端路径、工作目录和本地字体这类只属于当前设备的信息也会避免跟随同步传播，减少不同设备之间的干扰。

## 它适合哪些人

Termark 更适合经常和服务器打交道的人。

比如：

- 后端开发
- 运维和 SRE
- 自己维护服务器的独立开发者
- 经常处理客户服务器的技术支持
- 有多台云服务器、NAS、内网机器的人
- 已经在使用 NextTerminal，但也想要一个本地桌面终端工作台的人

如果你只偶尔登录一台服务器，它可能不是刚需。

但如果你经常在终端、SFTP、笔记、密码管理、跳板机配置、AI 对话和多个窗口之间切来切去，Termark 会更有价值。

它想做的不是一个“看起来功能很多”的工具，而是把远程工作中最常用、最容易被拆散的动作收在一起。
